ASCIISECSItem

This class representsa SECSItem with the SECS data type of A (ASCII), which is a "string" ofASCII characters.From the C# side this data type is handled as a C# string. Be careful to only use appropriate character sets (System.Text.Encoding.ASCII) or undesirable behavior may be manifested.

BinarySECSItem

This class represents a SECS item with a data type of B in an array form. In this case it is an array of zero or more 8-bit unsigned integers. The C# data type is byte[].

BooleanArraySECSItem

ListSECSItem

This class represents a SECSItem with the SECS data type of L, which is a list of SECSItems. From the C# side this data type is handled as a C# LinkedList<SECSItem>.

SECSItem

This is the base class for all of the SECSItems that are implemented in this library. Its purpose is to conceal the tedium of managing the information that will be needed for encoding/decoding the various SECSItems to/from the wire/transmission format that the transport layer(s) use.

SECSItemFactory

This is a static "factory" class that is used to create a SECSItem object from a byte[] that is in "wire/transmission" form. As you might expect the bytes contained within the byte[] must actually be in the form of those of an item in a SECS-II message.

SECSItemFormatCodeFunctions

A class containing a couple of helper methods useful when converting a SECSItem's format code to and from "wire / transmission" format.

SECSItemNumLengthBytes

This class represents the valid values for the number of length bytes for a SECSItem. This needed to be implemented in this manner instead of an enum because C# will automatically cast an enum to an int when used as a method argument. This may cause the compiler to use the wrong method if the signatures will match because of this.
