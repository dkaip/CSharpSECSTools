ASCIISECSItem

This class represents/implements a SECSItem with the SECS data type of A (ASCII), which is a "string" ofASCII characters.From the C# side this data type is handled as a C# string. Be careful to only use appropriate character sets (System.Text.Encoding.ASCII) or undesirable behavior may be manifested.

BinarySECSItem

This class represents/implements a SECSItem with the SECS data type of B(BINARY), which is an array of unsigned bytes.

BooleanArraySECSItem

This class represents/implements a SECSItem with the SECS data type of BO in an array form. In this case it is an array of zero or more 8-bit boolean values. From the C# side this data type is handled as a C# bool [].

In "wire/transmission" format all SECS items, with the exception of those with an item format code of L(List), are sent in an array form.For instance if an item is received which has an item format code of I4 (32-bit signed integer) and has a length of 8 you know that is it an array containing 2 4-byte signed integers. If it has a length of 0 you know it is an array with zero 4-byte signed integers. Likewise, If an item is received which has an item format code of U2 (16-bit unsigned integer) and has a length of 6 you know that is it an array containing 3 2-byte unsigned integers.

In practice, when only one item is received in the array (in the I4 case mentioned previously if the length was 4 instead of 8) it is handled and processed in a non array form. Hence I4SECSItem * vs I4ArraySECSItem, U2SECSItem vs U2ArraySECSItem, etc.

SECSItem

This is the base class for all of the SECSItems that are implemented in this library. Its purpose is to conceal the tedium of managing the information that will be needed for encoding/decoding the various SECSItems to/from the wire/transmission format that the transport layer(s) use.

SECSItemFactory

This is a static "factory" class that is used to create a SECSItem object from a byte[]. As you might expect the bytes contained within the byte[] must actually be in the form of those of an item in a SECS-II message.

SECSItemFormatCodeFunctions

A class containing a couple of helper methods useful when converting a SECSItem's format code to and from "wire / transmission" format.

SECSItemNumLengthBytes

This class represents the valid values for the number of length bytes for a SECSItem. This needed to be implemented in this manner instead of an enum because C# will automatically cast an enum to an int when used as a method argument. This may cause the compiler to use the wrong method if the signatures will match because of this.
